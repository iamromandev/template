# API Documentation

> **Base URL:** `http://localhost:8000/api/v1`
> **OpenAPI/Swagger:** `http://localhost:8000/docs` (auto-generated by FastAPI)

## Response Envelope

All API responses follow a consistent envelope:

### Success

```json
{
  "status": "success",
  "code": 200,
  "message": null,
  "data": { ... },
  "meta": null,
  "timestamp": "2026-07-01T12:00:00Z"
}
```

Pagination adds `meta`:

```json
{
  "status": "success",
  "code": 200,
  "data": [ ... ],
  "meta": {
    "page": 1,
    "page_size": 10,
    "total": 42,
    "total_pages": 5
  },
  "timestamp": "2026-07-01T12:00:00Z"
}
```

### Error

```json
{
  "status": "error",
  "code": 422,
  "message": "email: value is not a valid email address",
  "type": "validation_error",
  "details": null,
  "retry_able": false,
  "timestamp": "2026-07-01T12:00:00Z"
}
```

## Endpoints

### Health

#### `GET /health/check`

Checks application and database health.

**Response `200`:**
```json
{
  "data": {
    "version": "1.0.0",
    "db": {
      "status": "success",
      "version": "PostgreSQL 18.0 on x86_64-pc-linux-gnu..."
    }
  }
}
```

### Auth

#### `POST /users/auth/register`

Create a new user account.

**Request:**
```json
{
  "email": "user@example.com",
  "password": "Str0ng!Pass",
  "username": "johndoe"
}
```

**Response `201`:** `Success[UserSchema]` with `TokenSchema` embedded.

#### `POST /users/auth/login`

Authenticate and receive JWT tokens.

**Request:**
```json
{
  "login": "user@example.com",
  "password": "Str0ng!Pass"
}
```

`login` accepts either email or username.

**Response `200`:**
```json
{
  "data": {
    "access_token": "eyJ...",
    "refresh_token": "eyJ...",
    "token_type": "bearer",
    "expires_in": 1800
  }
}
```

#### `POST /users/auth/refresh`

Exchange a valid refresh token for a new token pair.

**Request:**
```json
{
  "refresh_token": "eyJ..."
}
```

**Response `200`:** `TokenSchema`

### Users

All endpoints require `Authorization: Bearer <access_token>`.

#### `GET /users/me`

Get the authenticated user's profile.

#### `PATCH /users/me`

Update the authenticated user's profile.

**Request:**
```json
{
  "username": "newusername",
  "email": "new@example.com"
}
```

### Metrics

#### `GET /metrics`

Prometheus metrics endpoint (OpenMetrics text format, excluded from schema).

## Authentication

1. Register or login to receive `access_token` + `refresh_token`.
2. Include `Authorization: Bearer <access_token>` in all authenticated requests.
3. When the access token expires (default 30 min), use `POST /auth/refresh` to get a new pair.

## Error Codes

| Status | Code | Meaning |
|--------|------|---------|
| 400 | BAD_REQUEST | Malformed request syntax |
| 401 | UNAUTHORIZED | Missing or invalid token |
| 403 | FORBIDDEN | Insufficient permissions |
| 404 | NOT_FOUND | Resource not found |
| 409 | CONFLICT | Resource already exists |
| 422 | UNPROCESSABLE_ENTITY | Validation error |
| 429 | TOO_MANY_REQUESTS | Rate limited |
| 500 | INTERNAL_SERVER_ERROR | Unexpected server error |
| 503 | SERVICE_UNAVAILABLE | Dependency unavailable |

## Pagination & Sorting

### Pagination

| Query param | Default | Description |
|-------------|---------|-------------|
| `page` | 1 | 1-indexed page number |
| `page_size` | 10 | Items per page (1–100) |

### Sorting

`?sort=-created_at,name`

Prefix a field with `-` for descending order. Multiple fields: comma-separated.
